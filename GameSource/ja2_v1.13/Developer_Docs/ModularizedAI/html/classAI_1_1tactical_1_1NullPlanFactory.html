Detailed Description

Concrete Factory. Generates the NullPlan, making a NPC do absolutely nothing.

The purpose of this factory is two-fold. Firstly, it is a good starting point for new plan factories; to use it as such, perform a

svn copy include/NullPlanFactory.h include/YourPlanFactory.h
svn copy src/NullPlanFactory.cpp src/YourPlanFactory.cpp

Remember to

  • Adjust the include guards
  • Adjust the string returned by get_name()
  • Add a line in PlanFactoryLibrary.cpp to create an instance of the new factory in the library's constructor; otherwise, it won't be accessible via the configuration file.

And secondly, it is a debugging tool. Change the configuration file to use this AI factory for a certain AI_INDEX to have said group do... nothing (e.g. to measure performance differences).

Used for delayed initialization of ressource-intensive initialization tasks.

Due to the design of the modularized AI, each factory's constructor must be called, even if the factory is not used. Therefore, memory- and cpu intensive tasks should not be performed in the constructor, but in this function instead. It will only be called if the factory is actually referenced in the AI.ini file.
